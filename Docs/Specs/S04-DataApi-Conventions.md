# 📑 S04: DataAPI Conventions & Attributes Specification

**Status:** 📝 Draft  
**Owner:** Engineering Team  
**Reviewers:** Community / Architects

## 1. Goal
Reduce boilerplate and verbosity when exposing database entities as REST APIs. This specification introduces "Convention over Configuration" patterns and attribute-driven discovery for the `Dext.DataApi` module.

## 2. Features

### 2.1. Global Defaults
Allow developers to define a "Base Configuration" for all DataAPIs in the application startup.

```pascal
DataApiOptions.SetDefaults(
  DataApiOptions
    .DbContext<TMyContext>
    .UseSnakeCase
    .UseSwagger
    .Paginate(50)
);
```

### 2.2. Attribute-Driven Discovery
Mark entities with the `[DataApi]` attribute to define routes and metadata directly on the model class.

```pascal
[DataApi('/api/customers', 'Customers')]
TCustomer = class(TEntity)
  ...
end;
```

### 2.3. Automatic Registration
Enable the discovery of all marked entities within a unit or the entire application.

```pascal
// Automatic discovery in the Program.pas
App.MapDataApisFromEntities<TMyContext>;
```

### 2.4. Smart Pluralization & Tagging
- If no route is provided in the attribute, Dext will use the class name (e.g., `TCustomer` -> `/api/customer`), **automatically stripping the "T" prefix**.
- If no Tag is provided, the Entity name (without "T") will be used as the Swagger Tag.

## 3. Technical Requirements

### 3.1. RTTI Scanner
- A high-performance scanner for class attributes in the application units.
- Cached results to avoid startup performance hits (integrated with `Dext.Core.Metadata`).

### 3.2. Mapping Logic
- Ensure that `MapDataApisFromEntities` respects existing manual registrations (Manual takes precedence over Automatic).
- Support for `[DataApiIgnore]` to explicitly hide an entity from discovery.

## 4. Acceptance Criteria
- [ ] A developer can expose 10 tables as REST endpoints with a single line of code in `Program.pas`.
- [ ] Changing a global default (e.g., `UseSnakeCase`) propagates to all automatic DataAPIs.
- [ ] Swagger documentation correctly displays tags and summaries for auto-generated endpoints.
