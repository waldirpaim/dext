## 1. Goal
Provide a seamless "Day 1" experience by allowing developers to generate boilerplate code for Web APIs and Entities using a CLI tool. The scaffolding must be intelligent enough to detect the Delphi version and architectural standards (Minimal API vs Controllers).

## 2. User Experience (CLI)

The CLI tool (`dext`) should support the following commands:

### 2.1. Project Initialization
```bash
dext new web [ProjectName] --type [minimal|mvc]
```
- **Templates**:
    - `minimal`: Generates a `Program.pas` with `WebApplication` and fluent routes.
    - `mvc`: Generates a folder structure with `Controllers/`, `Models/`, and `Services/`.

### 2.2. Development Productivity (dotpas inspired)
```bash
dext watch
```
- Automatically monitors `.pas` and `.inc` files.
- Triggers background compilation and restarts the server/application on save (Similar to `dotnet watch`).

```bash
dext dev-certs [trust]
```
- Manages local development certificates for HTTPS.
- Automates the installation of the CA in the Windows/Linux trust store.

### 2.3. Entity Generation (Database-First)
```bash
dext generate entity [TableName] --db [ConnectionName]
```
- Uses RTTI and DB Metadata to generate a Delphi unit with `[Table]` and `[Column]` attributes.
- Support for **Smart Properties** (`Prop<T>`) generation.

### 2.3. Endpoint Scaffolding
```bash
dext generate endpoint [EntityName] --type [minimal|controller]
```
- Creates CRUD endpoints for a specific entity.
- If `--type minimal`, appends to `Program.pas`.
- If `--type controller`, creates a new unit in `Controllers/`.

## 3. Technical Requirements

### 3.1. Template Engine
- Use the **Dext Template Processor** (modular and extensible).
- Support for **Web Stencils** syntax for Delphi 12.2+ environments.
- Fallback/Parity templates for legacy Delphi versions (VCL/Indy-focused).

### 3.2. Architecture
- Generated code must follow **Dependency Injection** patterns.
- Business logic should be separated into a `Service` layer if the project type is `mvc`.
- Proper use of `Record` for DTOs and `Class` for Entities.

## 4. Acceptance Criteria
- [ ] Command `dext new` creates a compilable project from scratch.
- [ ] `generate entity` correctly maps types (VarChar -> String, BigInt -> Int64).
- [ ] Generated Controllers include Swagger attributes (`[OpenApiSummary]`).
- [ ] Support for both Windows and Linux target output.
