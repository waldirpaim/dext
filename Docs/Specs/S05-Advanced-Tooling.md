# 📑 S05: Advanced Tooling (Wizards & Code Discovery) Specification

**Status:** 📝 Draft  
**Owner:** Engineering Team  
**Reviewers:** Community / Architects

---

## 1. Goal
Improve developer productivity by providing visual tools (Wizards) and code-discovery mechanisms inspired by JetBrains Rider and Visual Studio. Shift from CLI-only to an integrated IDE experience.

## 2. Features

### 2.1. Dext Explorer (IDE Integration)
- A specialized window (Dockable in Delphi IDE or standalone) to visualize:
    - **Entity Tree**: List of all `TEntity` classes detected in the project.
    - **Migration History**: View applied and pending migrations.
    - **API Routes**: Preview of all registered endpoints.

### 2.2. Migration Wizard
- **Code-First Parser**: A parser that reads `.pas` files to detect changes in Entities without requiring a full project compilation.
- **Visual Diff**: Preview the generated migration code before saving it to a unit.
- **Action Buttons**: `Add Migration`, `Update Database`, `Script Migration`.

### 2.3. Scaffolding UI
- A dialog to select templates and fill parameters (Target Unit, Base Class, etc.).
- **Live Template Editor**: Syntax-highlighted editor to customize Dext templates within the environment.

---

## 3. Technical Requirements

### 3.1. Parsing Engine
- Use a lightweight Delphi code parser (e.g., custom regex-based or simplified AST) to scan for `[Table]` and `[Column]` attributes.
- Support for detecting "Shadow Properties" and "Smart Properties".

### 3.2. IDE Bridge
- Implementation via **Open Tools API (OTA)** for deep integration.
- Fallback: A modern **WebView2** based standalone tool for developers using older Delphi versions or VS Code.

## 4. Acceptance Criteria
- [ ] Developer can add a new database column to a class and generate a migration with 3 clicks.
- [ ] No compilation is required to list existing entities in the Scaffolding Wizard.
- [ ] Templates can be overridden at the project level.
