# 📑 S02: Modernizer (gRPC & Protobuf) Specification

**Status:** 📝 Draft  
**Owner:** Engineering Team  
**Reviewers:** Community / Architects

---

## 1. Goal
Provide a high-performance, binary transport alternative to REST/JSON. The goal is to replace legacy connectivity technologies (DataSnap, RDW, RTC, RemObjects) with a modern, cross-platform standard based on gRPC and Protocol Buffers.

## 2. Core Components

### 2.1. Protobuf Integration
- Support for `.proto` file parsing and Delphi code generation.
- Native serialization/deserialization without intermediate JSON conversions.

### 2.2. TEntityDataSet DataProvider
- Bridge the VCL world with the gRPC world.
- A specialized **DataProvider** that allows a `TEntityDataSet` in a Desktop client to fetch/update data from a gRPC service seamlessly.

### 2.3. Service Transformation
- Ability to expose Dext Entities as gRPC Services with minimal configuration (using attributes or fluent mapping).
- Support for **Unary**, **Server Streaming**, and **Bi-directional Streaming**.

---

## 3. Technical Requirements

### 3.1. Connectivity & Transport
- **Phased Implementation**:
    - **Phase 1 (Legacy Replacement)**: Wrapper over DCS (Delphi Cross Socket) for rapid prototyping and gRPC compatibility validation.
    - **Phase 2 (Infrastructure Independence)**: Implementation of a **Native Dext Server Engine** using **IOCP (Windows)** and **Epoll (Linux)** for zero-dependency high-performance HTTP/2 support.
- **gRPC-Web**: Built-in proxy/support to allow browser-based clients to consume Dext gRPC services directly.

### 3.2. Mapping & Serialization
- Automatic mapping between Dext Entities and Protobuf Messages (Inspired by `protobuf-net` code-first approach).
- Handling of specialized types (GUID v7, Nullables, Blobs).

## 4. Acceptance Criteria
- [ ] A VCL project can populate a `TDBGrid` via gRPC using a `TEntityDataSet`.
- [ ] Performance must be at least 3x faster than REST/JSON for large datasets on the Native Engine.
- [ ] Compatibility with non-Delphi clients (Python/Node.js) using the generated `.proto` files.
