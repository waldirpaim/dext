# Roadmap

View the future of the Dext Framework.

## 🔴 Phase 1: Stability & RC 1.0 (Current)
Focus on core performance, standard feature completion, and documentation.

- [x] **Path Versioning**: `/v1/api` support.
- [x] **OAuth2 Client Credentials**: Enterprise machine-to-machine auth.
- [x] **Web Object Tracking**: Robust memory management in request pipeline.
- [x] **DI Lifecycle Stabilization**: Resolving scope and destruction order.
- [x] **Professional Documentation**: Centralized Book with parity.

## 🟡 Phase 2: Ecosystem & Developer Experience
Enhancing the developer workflow and adding common utilities.

- [ ] **Global Filters**: Apply Action Filters to all controllers at once.
- [ ] **Advanced Validation**: Fluent validation integration for entities.
- [ ] **Background Workers (Queue)**: Native support for background processing with retries.
- [ ] **Logging Providers**: Out-of-the-box support for ELK, Seq, and Sentry.
- [ ] **IDE Plugin Evolution**: Enhanced visual designers and scaffolding.

## 🟢 Phase 3: Future & Innovation
Long-term vision for the framework.

- [ ] **Desktop MVU Engine**: Model-View-Update pattern for Dext.UI.
- [ ] **JSON UTF-8 Parser**: Zero-allocation, high-performance JSON engine.
- [ ] **Cloud-Native Adapters**: Direct integration with AWS Lambda and Azure Functions.
- [ ] **Real-Time Hubs Evolution**: Horizontal scaling for WebSockets.
